name: fix-review-findings
description: Address all issues and gaps found by reviews, regardless of priority
---

Address all issues and gaps found by reviews, regardless of priority, if they can be addressed now.

The only acceptable reason to skip a finding is a genuine architectural constraint that would require a separate design effort — not "it's low priority" or "the churn isn't worth it." If skipping a finding, state the constraint explicitly and get user confirmation before proceeding.
