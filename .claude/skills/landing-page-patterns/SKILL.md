## Page Type Taxonomy

### Lead Generation
**Goal**: Capture contact information (email, phone, form submission)
**Key sections**: Hero with value prop → pain points → solution → social proof → form/CTA → FAQ
**CTA strategy**: Single primary action (form submission). Repeat CTA 2-3 times. Keep form short (3-5 fields max). Consider multi-step forms for complex captures.
**Above the fold**: Headline + subheadline + primary CTA or short form
**Best for**: B2B SaaS, professional services, webinar signups, gated content

### Product Launch
**Goal**: Drive awareness, waitlist signups, or pre-orders
**Key sections**: Hero with product visual → key features (3-4) → deep dive → social proof → pricing (if applicable) → CTA
**CTA strategy**: Primary CTA changes based on stage: "Join Waitlist" (pre-launch) → "Get Started" / "Buy Now" (launch). Feature the product visually.
**Above the fold**: Product image/screenshot + compelling headline + primary CTA
**Best for**: New product releases, feature announcements, app launches

### Event / Webinar Signup
**Goal**: Drive registrations
**Key sections**: Hero with event details → speakers/hosts → agenda → social proof (past event stats) → registration CTA → FAQ
**CTA strategy**: Registration form or button. Include urgency (date, limited spots). Repeat CTA after each value section.
**Above the fold**: Event name + date/time + primary speaker + register CTA
**Best for**: Conferences, webinars, workshops, product demos

### App Download
**Goal**: Drive app store visits / downloads
**Key sections**: Hero with device mockup → key features → how it works (3 steps) → testimonials/ratings → app store badges
**CTA strategy**: App store buttons (iOS + Android). Show device mockups. Include ratings/review counts.
**Above the fold**: Device mockup + headline + app store badges
**Best for**: Mobile apps, desktop software

### Brand Awareness / Storytelling
**Goal**: Build brand perception, not immediate conversion
**Key sections**: Hero (visual-heavy) → brand story → values/mission → impact metrics → team/culture → soft CTA
**CTA strategy**: Soft CTAs (Learn More, Explore, Follow Us). Focus on engagement over conversion.
**Above the fold**: Bold visual + brand statement
**Best for**: Corporate sites, non-profits, brand refreshes

### Comparison / Competitive
**Goal**: Win against a specific competitor or category
**Key sections**: Hero positioning against status quo → comparison table → unique advantages → testimonials from switchers → migration CTA → FAQ
**CTA strategy**: "Switch Now", "Compare Plans", "See How We're Different". Address switching costs directly.
**Above the fold**: Headline addressing the pain of the current solution + comparison hook
**Best for**: Competitive markets, migration campaigns

## Section Pattern Library

### Hero Variants

**Centered Hero**
- Headline, subheadline, CTA centered
- Full-width background image or color
- Best for: simple messages, brand awareness
- Risk: can feel generic

**Split Hero**
- Text on one side, image/visual on the other
- Best for: product-focused pages where showing the product matters
- Typically text left (55%), image right (45%)

**Video Hero**
- Background video or embedded player
- Best for: experiential brands, product demos
- Risk: performance impact, autoplay issues on mobile

**Minimal Hero**
- Large headline, minimal supporting elements
- Best for: high-confidence brands, statement pages
- Works when the headline alone is compelling enough

### Value Proposition Patterns

**Icon Grid** (3 or 4 columns)
- Icon + title + short description per item
- Best for: feature overviews, benefit summaries
- Mobile: stack to single column

**Alternating Image/Text**
- Image left + text right, then flip
- Best for: feature deep dives, storytelling
- 2-4 alternating sections max before fatigue

**Cards**
- Equal-height cards in a grid
- Best for: plans, team members, use cases
- Mobile: horizontal scroll or stack

### Social Proof Patterns

**Logo Strip**
- Row of client/partner logos
- Best for: B2B, trust building
- 5-8 logos, grayscale for consistency

**Testimonial Cards**
- Quote + name + photo + title
- Best for: B2C, emotional connection
- Include specific results when possible ("increased revenue 40%")

**Stats Bar**
- 3-4 large numbers with labels
- Best for: scale/impact (users, revenue, uptime)
- Animate on scroll for engagement

### CTA Patterns

**Inline CTA**
- Button within a content section
- Repeat after every 2-3 sections

**Sticky CTA**
- Fixed bar at top or bottom (mobile especially)
- Best for: long pages where the main CTA scrolls out of view

**End CTA Block**
- Dedicated full-width section near the bottom
- Headline restating value + primary button
- Last chance before footer

## Content Hierarchy Principles

1. **Headline**: Address the visitor's primary desire or pain point. Not the brand name.
2. **Subheadline**: Explain how you deliver on the headline's promise. One sentence.
3. **Social proof early**: Logos or a stat bar right after the hero builds trust before asking for engagement.
4. **Features → Benefits**: Don't just list features. Connect each to a user outcome.
5. **Address objections**: FAQ or comparison sections handle the "yes, but..." thoughts.
6. **Final CTA**: Restate the value proposition in different words. Make the action obvious.

## Responsive Strategy Principles

- **Mobile**: Single column, shorter headlines, larger touch targets (min 44px), consider hiding secondary content
- **Tablet**: Two-column grids where appropriate, slightly reduced spacing
- **Desktop**: Full layouts, maximum visual impact, multi-column grids
- **CTA on mobile**: Consider sticky bottom bar if the primary CTA is above the fold only on desktop
- **Images on mobile**: Simplify or crop differently (art direction), not just scale down
- **Navigation on mobile**: Hamburger menu, keep it minimal (5-7 items max)
